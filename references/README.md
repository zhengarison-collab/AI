# Portfolio reference materials

These files were prepared from the images, short video, and resume Terry supplied for the personal portfolio build.

## Contents

### Visual direction

- `visual/homepage-hero-reference-01.svg` — first dark editorial homepage composition with a cybernetic floral portrait.
- `visual/homepage-hero-reference-02.svg` — alternate homepage composition and headline treatment.
- `visual/cyber-floral-portrait-reference.svg` — vertical character, pose, lighting, and visual-language reference.

The SVG files contain embedded raster previews, so they remain viewable and self-contained after the repository is cloned.

### Motion direction

- `motion/portfolio-motion-contact-sheet.svg` — a sequence of key frames extracted from the supplied motion reference. Use it to infer pacing, framing, transitions, and cinematic movement; do not treat it as production footage.

### Resume source

- `resume/zheng-chao-resume-2024.md` — structured transcription of the supplied two-page resume for About, Experience, and Resume pages.
- `resume/zheng-chao-resume-preview.svg` — low-resolution two-page layout preview.

## Implementation guidance for Codex

1. Treat the visual files as references rather than final production assets.
2. Preserve the dark editorial composition, restrained typography, cyan-pink bio-cybernetic lighting, floral integration, and cinematic depth.
3. Use the resume Markdown as the authoritative content source, but rewrite it for a modern international portfolio rather than reproducing the old resume layout.
4. Do not publish phone numbers, personal email addresses, age, or other private resume fields on the website unless the repository owner explicitly requests it.
5. Replace the embedded previews with final high-resolution assets before production deployment when those files become available inside the repository.

The reference derivatives are intentionally lightweight so cloud Codex can inspect them directly from the repository.
